# SM4 加密 TCP 通信系统

本项目实现了基于国密 SM4 加密算法的安全 TCP 通信系统，支持客户端与服务器之间的加密数据传输。

## 项目概述

该系统结合了 RSA 非对称加密和 SM4 对称加密算法：
- **RSA**: 用于安全地交换 SM4 密钥（客户端使用公钥加密，服务器使用私钥解密）
- **SM4**: 国家密码管理局发布的分组密码算法，用于实际的数据加密传输

## 功能特点

- ✅ **安全密钥交换**: 使用 RSA 2048 位密钥对安全交换 SM4 会话密钥
- ✅ **国密算法支持**: 完整实现 SM4 分组密码算法（128 位密钥，128 位分组）
- ✅ **双向通信**: 支持客户端与服务器之间的实时加密聊天
- ✅ **多平台支持**: 提供命令行版本和图形界面版本
- ✅ **详细日志**: 完整的调试信息输出，便于理解加密流程

## 编译说明

### 编译依赖

- GCC 编译器
- OpenSSL 开发库

### 编译命令

```bash
# 编译服务器
gcc -o server server.c sm4.c -lssl -lcrypto -lpthread

# 编译客户端  
gcc -o client client.c sm4.c -lssl -lcrypto -lpthread
```

## 使用方法

### 方法一：命令行模式

#### 启动服务器
```bash
./server
```
服务器将在端口 8888 监听连接。

#### 启动客户端
```bash
./client <服务器IP地址>
```
例如连接本地服务器：
```bash
./client 127.0.0.1
```

### 方法二：图形界面模式

运行 Python GUI 客户端：
```bash
python chat_gui.py
```

在界面中选择运行模式：
- **服务端（Server）**: 启动 TCP 服务器
- **客户端（Client）**: 连接指定 IP 的服务器

## 通信流程

1. **客户端启动**:
   - 生成随机的 16 字节 SM4 密钥
   - 使用服务器的 RSA 公钥加密 SM4 密钥
   - 将加密后的密钥发送给服务器

2. **服务器处理**:
   - 接收加密的 SM4 密钥
   - 使用 RSA 私钥解密获得原始 SM4 密钥
   - 建立安全通信通道

3. **数据传输**:
   - 所有后续通信数据都使用 SM4 算法加密/解密
   - 支持双向实时消息传输

## 安全说明

- **密钥管理**: 项目包含示例密钥文件（`pub.pem` 和 `priv.pem`），实际部署时应生成新的密钥对
- **密钥长度**: SM4 使用 128 位密钥，RSA 使用 2048 位密钥，符合当前安全标准
- **随机数生成**: 客户端使用 `/dev/urandom` 生成高质量随机密钥

## 配置参数

代码中的主要配置参数：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `PORT` | 8888 | TCP 通信端口 |
| `BUF_SIZE` | 1024 | 缓冲区大小 |
| `SM4_KEY_LEN` | 16 | SM4 密钥长度（字节） |
| `RSA_KEY_SIZE` | 2048 | RSA 密钥长度（位） |
